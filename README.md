# 🧭 中津くらしナビ

大分県中津市周辺で暮らす人が「困った時にすぐ見る」ための生活便利サイトです。
市役所・病院・薬局・ATM・コインランドリー・安いご飯・相談窓口など、暮らしに関わる実在の施設をカテゴリやキーワードからすぐ探せます。

このサイトは **HTML・CSS・JavaScriptだけ** で作られた「静的サイト」です。
React・Vue・Next.jsなどのフレームワークは使っていないため、GitHub Pagesでそのまま公開できます。

> ⚠️ **掲載情報は変更されている場合があります。お出かけ前・利用前に、各施設の公式情報または電話でご確認ください。**
> 営業時間・電話番号などが未確認の施設には「要確認」バッジを表示しています。

## 3. 実在施設を掲載する時の注意（いちばん大事）

このサイトは実在の施設名を掲載します。**間違った情報は、見た人に実害を与える可能性があります**（閉まっている病院に行ってしまう、違う番号に電話してしまう等）。次のルールを必ず守ってください。

### 🚫 やってはいけないこと

1. **架空の営業時間・電話番号・定休日を書かない。** 「たぶんこうだろう」で書くのは禁止です。
2. **記憶やうわさだけで書かない。** 「前に行った時は18時までだった」も、公式情報で確認するまでは書きません。
3. **確認していない施設に `verified: true` を付けない。**
4. まとめサイト・口コミサイトの情報を**そのまま**転記しない（変更が反映されていないことが多いため）。

### ✅ 確認できない時のルール

- 確認できない項目は **空文字 `""` のまま** にする → 画面には自動で「要確認」と表示されます。
- 施設の存在自体は確かだが詳細が不明な場合は `verified: false` にする → 「⚠ 要確認」バッジが付きます。
- 営業時間が空欄の施設は、「営業中っぽい施設だけ表示」の判定対象から自動的に外れます（誤った「営業中」表示を防ぐため）。

## 4. 情報源の確認方法

`verified: true` にしてよいのは、次のような**一次情報**で確認できた場合だけです。

| 優先度 | 情報源 | 例 |
|---|---|---|
| ◎ | 施設の公式サイト・公式SNS | 病院の公式ページの「外来受付時間」など |
| ◎ | 行政の公式サイト | 中津市公式サイト（city-nakatsu.jp）、国民生活センターなど |
| ○ | 電話で施設に直接確認 | 確認した日付をsourceMemoに残す |
| △ | 地図アプリ・ポータルサイト | 参考にはなるが、これ**だけ**でverified:trueにしない |
| ✕ | 個人ブログ・古い口コミ | 情報源にしない |

確認したら、`sourceMemo` に **「いつ・どこで確認したか」** を必ず書きます。

```
sourceMemo: "2026年7月 公式サイトで確認"
```

情報は時間が経つと古くなります。**半年〜1年に1回は確認済み施設を見直す**ことをおすすめします。見直しが難しくなったら、`verified: false` に戻すのも誠実な運用です。

## 5. 店舗情報の追加方法

`script.js` の `const facilities = [ ... ]` 配列に、以下の形式でオブジェクトを追加します。

```js
{
  id: 33,                     // 他と重複しない数字（一番大きい番号+1）
  name: "施設の正式名称",
  category: "薬局",           // 下の「カテゴリ一覧」から1つ（1文字も違わず一致させる）
  description: "施設の簡単な説明",
  address: "大分県中津市〇〇", // 確認できた範囲だけ書く
  phone: "",                  // ★確認できるまで空欄のまま
  hours: "",                  // ★確認できるまで空欄のまま（確認後は "09:00〜18:00" 形式）
  closed: "",                 // ★確認できるまで空欄のまま
  mapUrl: "",                 // 空欄なら施設名＋住所で自動生成される
  officialUrl: "",            // 公式サイトがあればURLを入れる（「公式情報を見る」ボタンになる）
  sourceMemo: "営業時間は公式サイトで要確認",
  tags: ["タグ1", "タグ2"],
  memo: "",                   // 利用者向けのひとこと（任意）
  verified: false             // ★公式情報で確認できるまで false のまま
}
```

追加のポイント：

- 配列内の他のオブジェクトと同じように、**末尾のカンマ「,」** を忘れない（エラーで画面が動かなくなる最多原因）。
- `hours` は `09:00〜18:00` の形式で書くと、営業中判定が自動で動きます。`24時間` と書くと常に営業中扱いです。
- `closed` に「日曜」など曜日の漢字を含めると、その曜日は「本日定休日」と表示されます。
- チェーン店で店舗が特定できない場合は、名前を「〇〇（中津市内 各店舗）」とし、公式サイトの店舗検索へ誘導する形にします（既存データに例があります）。

### カテゴリ一覧（`category` に指定できる文字列）

市役所・手続き ／ 病院・歯医者 ／ 薬局 ／ ATM・銀行 ／ コインランドリー ／ 安いご飯 ／ 弁当・惣菜 ／ 雨の日に行ける場所 ／ 釣り場 ／ 困った時の相談先

## 6. 間違った情報を載せないためのルール（まとめ）

1. **書く前に確認、確認できなければ空欄** — 空欄は自動で「要確認」表示になるので、空欄は恥ずかしいことではありません。
2. **verified: true は一次情報＋sourceMemo記入とセット** — どちらか欠けたらfalseのまま。
3. **利用者への注意書きを消さない** — サイト上部・下部の「掲載情報は変更されている場合があります。お出かけ前・利用前に、各施設の公式情報または電話でご確認ください。」は必ず残す。
4. **指摘を受けたらすぐ直す** — 修正依頼が来たら、公式URLで確認のうえ修正。確認できなければその項目を空欄に戻す。
5. **相談窓口・医療は特に慎重に** — 間違いの影響が大きいカテゴリです。全国共通番号（188・189・#9110など）以外は、必ず公式ページで確認してから載せる。

## 7. GitHubでデータを更新する方法

公開後の更新は、すべてブラウザ上のGitHub操作だけで完結します。

### 施設を追加・修正する

1. GitHubで自分のリポジトリを開く
2. ファイル一覧から `script.js` をクリック
3. 右上の **えんぴつマーク（Edit this file）** をクリック
4. `const facilities = [` の中を編集（追加は配列の末尾、修正は該当施設のオブジェクトを書き換え）
5. 画面右上の **「Commit changes...」** をクリック
6. 変更内容のメモ（例：「宝来軒の営業時間を公式確認して追加」）を書いて **「Commit changes」**
7. 1〜2分待つと公開ページに反映されます

### 更新時のチェックリスト

- [ ] カンマの付け忘れ・付けすぎがないか（編集画面で赤くならないか確認）
- [ ] `category` の文字列がカテゴリ一覧と完全一致しているか
- [ ] 確認していない項目を埋めていないか（空欄はそのままでOK）
- [ ] `verified: true` にした場合、`sourceMemo` に確認日と情報源を書いたか
- [ ] 反映後、スマホで実際に開いてカードの表示を確認したか

### 失敗した時（画面が真っ白になった等）

リポジトリの `script.js` ページで「History」→ 直前のバージョンを開き、内容をコピーして貼り直せば元に戻せます。焦らなくて大丈夫です。

## 8. GitHub Pagesで公開する方法（未公開の場合）

1. GitHubで新しいリポジトリを作成（Public）
2. 4ファイルをリポジトリ直下にアップロード（「Add file」→「Upload files」）
3. **Settings → Pages** を開く
4. Source を **Deploy from a branch**、Branch を `main` / `(root)` にして **Save**
5. 数分後、`https://ユーザー名.github.io/リポジトリ名/` で公開されます

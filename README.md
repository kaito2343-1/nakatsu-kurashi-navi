# 🧭 中津くらしナビ

大分県中津市周辺で暮らす人が「困った時にすぐ見る」ための生活便利サイトです。
病院・薬局・ATM・コインランドリー・安いご飯・相談窓口など、暮らしに関わる施設をカテゴリやキーワードからすぐ探せます。

このサイトは **HTML・CSS・JavaScriptだけ** で作られた「静的サイト」です。
React・Vue・Next.jsなどのフレームワークは一切使っていないので、初心者の方でも中身を読んだり書き換えたりしやすい作りになっています。

> ⚠️ **現在はサンプルデータで動いています。** 掲載されている施設名・電話番号・営業時間は実在の情報ではありません。サイト上部・下部にもその旨の注意書きを表示しています。

## 1. サイト概要

| 項目 | 内容 |
|---|---|
| サイト名 | 中津くらしナビ |
| コンセプト | 中津で困った時にすぐ見る生活便利サイト |
| 対象ユーザー | 中津市周辺に住んでいる一般の方 |

### 主な機能

- 🔎 キーワード検索（施設名・説明・タグから検索、入力中もリアルタイムで絞り込み）
- 🗂️ カテゴリ絞り込み（10カテゴリ・横スクロール・件数表示・**カテゴリごとの色分け**つき）
- 🕒 「営業中っぽい施設だけ表示」チェックボックス（現在時刻から簡易判定）
- ⭐ お気に入り登録（localStorageに保存。追加・解除時に**アニメーションとトースト通知**で反応）
- 🗺️ Googleマップで開く／📞 電話するボタン
- 📩 掲載依頼ボタン（押すと仮のお問い合わせ案内が表示されます）
- 🏷️ サンプルデータのカードには「サンプル」バッジを自動表示
- 📱 スマホ幅（最大430px）を基準にしたカード型レイアウト

## 2. ファイル構成

```
.
├── index.html   … ページの骨組み（HTML）
├── style.css    … 見た目・デザイン（CSS）
├── script.js    … 施設データと検索・お気に入りなどの動き（JavaScript）
└── README.md    … このファイル
```

**⚠️ 重要：4つのファイルは必ず「同じフォルダ」に置いてください。**
`index.html` は同じフォルダにある `style.css` と `script.js` を読み込む作りになっています。場所やファイル名がずれると、CSSが効かない・カードが表示されないといった不具合になります。

## 3. パソコンで動かして確認する

インストール作業は不要です。

1. 4つのファイルを同じフォルダに置く
2. `index.html` をダブルクリックして、ブラウザ（Chrome・Edge・Safariなど）で開く

初期表示で全21件の施設カードが並んでいれば正常に動いています。

### うまく表示されない時のチェックリスト

| 症状 | 原因の可能性 | 確認すること |
|---|---|---|
| 文字が大きいだけで飾りが無い | CSSが読み込めていない | `style.css` が `index.html` と同じフォルダにあるか。ファイル名が正確に `style.css` か（`Style.css` や `style.css.txt` はNG） |
| カードが1件も出ない | JSが読み込めていない／エラー | `script.js` が同じフォルダにあるか。ブラウザでF12キー →「コンソール」に赤いエラーが出ていないか |
| ボタンを押しても反応しない | JSのエラー | 同上。エラーの行番号を見るとどこで止まっているか分かります |
| 古い表示のまま変わらない | ブラウザのキャッシュ | Ctrl+Shift+R（Macは Cmd+Shift+R）で強制再読み込み |

※ Windowsで拡張子（.css / .js）が見えない場合は、エクスプローラーの「表示」→「ファイル名拡張子」にチェックを入れると確認できます。

## 4. データの追加・修正方法

`script.js` を開き、`const facilities = [ ... ]` という配列の中に、以下の形式でオブジェクトを追加してください。

```js
{
  id: 22,                // 他のidと重複しない数字にする（一番大きい番号+1でOK）
  isSample: true,        // サンプルデータなら true。実在情報に差し替えたら false にする
  name: "施設名",
  category: "病院・歯医者", // 下記「使えるカテゴリ一覧」から1つを選ぶ
  description: "施設の簡単な説明文",
  address: "住所",
  phone: "0979-00-0000", // 電話番号が無い場合は "" のままでOK
  hours: "09:00〜18:00",  // 「24時間」と書くと24時間営業として扱われます
  closed: "日曜",         // 定休日が無い場合は "なし" と書く
  mapUrl: "https://www.google.com/maps/search/?api=1&query=中津市 施設名",
  tags: ["タグ1", "タグ2"],
  memo: "補足情報（無ければ空文字 \"\" でOK）"
}
```

書き終えたら、配列の中の他のオブジェクトと同じように **カンマ「,」で区切る** のを忘れないようにしてください（カンマ忘れはエラーで画面が動かなくなる一番多い原因です）。

### 「サンプル」バッジについて

- `isSample: true` の施設カードには、右上に「サンプル」というバッジが自動で表示されます。
- 実在の正確な情報に差し替えたら、`isSample: false` に変える（または行ごと消す）とバッジが消えます。
- **確認が取れていない情報のうちは `true` のままにしておくことを強くおすすめします。**

### 使えるカテゴリ一覧（`category`に指定できる文字列）

| カテゴリ名 | カードの色 |
|---|---|
| 病院・歯医者 | 赤 |
| 薬局 | 緑 |
| ATM・銀行 | 青 |
| コインランドリー | 水色 |
| 安いご飯 | オレンジ |
| 弁当・惣菜 | からし色 |
| 雨の日に行ける場所 | 紫 |
| 釣り場 | 海の青 |
| 市役所・手続き | 深緑 |
| 困った時の相談先 | 朱色 |

※ カテゴリ名は `script.js` の先頭にある `CATEGORIES` の一覧と1文字も違わず一致させてください。

### 新しいカテゴリそのものを追加したい場合

1. `script.js` の `CATEGORIES` 配列の末尾に1行追加する
   ```js
   { key: "美容院・理容室", icon: "💇" },
   ```
2. `style.css` の「カテゴリごとの色分け」の部分に、続きの番号で色を1行追加する
   ```css
   .cat-10 { --cat-color: #a0527d; --cat-color-soft: #f8e9f1; }
   ```

### 「営業中っぽい」判定のしくみ

- `hours` に `24時間` と書くと、常に「営業中」として扱われます。
- `hours` に `09:00〜18:00` のように書くと、現在時刻がその範囲内かどうかを自動で判定します。
- `closed` に「日曜」など曜日の漢字を入れておくと、その曜日は「本日定休日」と表示されます。
- あくまで文字列からの簡易的な推測なので、実際の営業状況と異なる場合があります。

## 5. GitHub Pagesで公開する方法（初心者向け・くわしい版）

### 手順0：準備するもの

- GitHubの無料アカウント（[github.com](https://github.com/) の「Sign up」から作成できます。メールアドレスがあればOK）
- この4つのファイル（index.html / style.css / script.js / README.md）

### 手順1：リポジトリ（ファイル置き場）を作る

1. GitHubにログインし、画面右上の「＋」→「**New repository**」をクリック
2. 「Repository name」に好きな名前を入力（例：`nakatsu-kurashi-navi`）
   - この名前が公開URLの一部になります（後で変えるのは少し手間なので、最初に決めましょう）
3. 「**Public**」が選ばれていることを確認（Privateだと無料プランでは公開できません）
4. 他の項目はそのままで「**Create repository**」をクリック

### 手順2：ファイルをアップロードする

1. 作成したリポジトリの画面で「**uploading an existing file**」というリンク（または「Add file」→「Upload files」）をクリック
2. 4つのファイルを画面にまとめてドラッグ＆ドロップ
   - **フォルダごとではなく、ファイルを直接** ドロップしてください。フォルダの中に入るとURLが変わってしまいます
3. 下の緑色の「**Commit changes**」ボタンをクリック
4. リポジトリのトップに4つのファイルが並んでいればOK

### 手順3：GitHub Pagesをオンにする

1. リポジトリ上部のタブから「**Settings**」をクリック
2. 左側のメニューから「**Pages**」をクリック
3. 「Build and deployment」の「Source」が「**Deploy from a branch**」になっていることを確認
4. 「Branch」のプルダウンで「**main**」を選び、隣のフォルダは「**/(root)**」のまま「**Save**」をクリック

### 手順4：公開URLを確認する

1. 1〜3分ほど待ってから、同じPagesの設定画面を再読み込みする
2. 上部に「Your site is live at `https://ユーザー名.github.io/リポジトリ名/`」と表示されたら公開完了です
3. そのURLをスマホでも開いて、表示を確認しましょう

### 手順5：あとから内容を更新するには

1. リポジトリのファイル一覧で、修正したいファイル（例：`script.js`）をクリック
2. 右上のえんぴつマーク（Edit）をクリックして編集
3. 「Commit changes」で保存すると、1〜2分後に公開ページにも反映されます

### よくあるつまずき

| 症状 | 対処 |
|---|---|
| 404 Not Found になる | 公開まで数分かかることがあります。また `index.html` という名前のファイルがリポジトリ直下にあるか確認 |
| 公開ページだけCSSが効かない | style.css / script.js が index.html と同じ階層（直下）にあるか確認 |
| 更新したのに変わらない | スマホ・PCともに強制再読み込み（Ctrl+Shift+R）。それでもダメなら数分待つ |

## 7. 用語がわからない時のミニ用語集

| 用語 | かんたんな説明 |
|---|---|
| リポジトリ | GitHub上でファイルをまとめて保管する「フォルダ」のようなもの |
| GitHub Pages | GitHubに置いたファイルを、そのままWebサイトとして公開してくれる無料の仕組み |
| コミット | GitHubに変更を保存すること。「Commit changes」ボタン＝保存ボタン |
| localStorage | ブラウザの中にデータを保存しておく仕組み。お気に入り機能に使用しています |
| トースト | 画面の下からふわっと出てくる小さな通知メッセージのこと |
| コンソール | ブラウザでF12キーを押すと出てくる開発者用画面。エラーが赤い文字で表示されます |
